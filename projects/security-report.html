Web Security Report

A vulnerability analysis focused on risk, cause, and repair.

Threat modelingWeb securityRisk explanationMitigation

Overview

This artifact came from cybersecurity work where the goal was to analyze a vulnerable system instead of only building a feature. I treated the report as a way to explain how a weakness appears, how it could be used, and how the design could be improved.

Problem

A system can look finished and still be fragile. Security work asks a different question: what happens when someone does not use the system the way the designer expected?

Process

This is the basic path I followed while developing or analyzing the artifact.

01Identified vulnerable behavior in a small web application.
02Traced how the weakness could affect users or data.
03Separated the technical cause from the visible symptom.
04Wrote mitigation notes in plain language instead of hiding behind jargon.

Value

This artifact adds balance to my eP. It shows that I do not only want to build software; I want to question what I build and understand where it can fail.

Reflection

The main lesson was humility. A working feature is not automatically a safe feature.
