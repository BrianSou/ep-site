UVA ClubHub

A database-backed catalog for UVA student organizations.

FlaskMySQLE-R designSQLTeam project

Overview

UVA ClubHub was a database systems project built around a real campus problem: student organizations are useful, but they can be hard to search and compare. My team designed a web app where students could browse clubs, save favorites, request membership, and come back to their saved data.

Problem

The visible website is only half the project. The more important problem was how to structure the data behind students, clubs, events, announcements, saved clubs, and membership requests so the app could answer real questions without turning into a messy set of tables.

Process

This is the basic path I followed while developing or analyzing the artifact.

01Mapped the main users: students, club officers, and admins.
02Built an E-R model for users, clubs, memberships, events, announcements, and saved clubs.
03Translated the design into normalized tables and SQL queries.
04Connected the database logic to user-facing screens and officer workflows.

Value

This is a strong portfolio artifact because it shows both product thinking and systems thinking. A club catalog sounds simple, but the database choices decide whether the app stays organized as features grow.

Reflection

The project made me more patient. Database work rewards slow decisions early because weak structure becomes expensive later.
