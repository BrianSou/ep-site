Reliable File Transfer

A networks assignment about making communication survive failure.

SocketsPacketsReliabilityProtocols

Overview

This project focused on the part of computing users usually never see: how data moves between machines. The artifact shows a file transfer process designed to keep working even when messages are dropped, delayed, or received out of order.

Problem

Sending a file sounds simple until the connection is unreliable. The program has to track what was sent, what was received, what needs to be retried, and how the receiver should reconstruct the final file.

Process

This is the basic path I followed while developing or analyzing the artifact.

01Split data into packets with ordering information.
02Handled acknowledgments and retransmission logic.
03Tested edge cases where messages were delayed or missing.
04Explained the communication flow with a sender-receiver diagram.

Value

This artifact supports the systems side of my professional narrative. It shows that I care about what happens underneath the interface, not just what appears on the screen.

Reflection

The useful lesson was that reliability is designed. It is not a default property of software.
